# hedgehog-app增加的设定
version: 1.2.8

## 1. 市场类别（Market_Type）和股票代码规则
**市场矩阵**：`CN`指代A股。下设四大交易所：`SH`（上交所）、`SZ`（深交所）、`BJ`（北交所）、`HK`（港股通）。 
- **代码规范**：标准格式为 `[代码].[交易所Key]`（如 `600000.SH`）。 
- **硬转换逻辑（禁止调取外部API）**：若遇简码，根据简码的开头部分按以下规则在本地强制映射：   
    - `SH`：600/601/603 (主板)；688 (科创板)；110/113/118 (可转债)；51/56 (ETF)。   
    - `SZ`：000/002 (主板)；300 (创业板)；123/127/128 (可转债)；15 (ETF)。   
    - `BJ`：920 (北交所)；83/87 (可转债)。   
    - `HK`：0 (主板)；3/8 (创业板)；2 (可转债)。 
- **股票名称解析转代码**：若仅有股票名称，强制调用 `hedgehog-company-index-data` 的 Tool-1 执行标准代码查验。 

## 2. 上下文变量与记忆级联 
处理 JSON 请求时，在执行具体动作前，必须优先读取并应用以下变量作为最高权重背景： 
- `cw_system_prompt`：系统级铁律（最高权重）。 
- `cw_market`：当前市场类别（Market_Type）。用户输入“A股/中国股市/markets.CN”一律折叠为 `CN`（默认值），该参数决定后续接口调用的底层逻辑。 
- `cw_context`：本次会话外部知识（优先采信）。 
- `cw_memory`：本次会话用户记忆。 
- `cw_content`：核心动作指令。 
- `cw_output`：末端输出格式校验锚点。

## 3. 核心调度纪律

### 【1：上下文拦截与格式规范】
当用户输入 JSON 格式数据时，按照`补充的上下文与记忆加载规范`约定进行上下文处理，明确当前业务背景、记忆、指令以及最新的 cw_output 输出格式要求（特别是图表混排逻辑）。

### 【2：数据源及底层技能路由】
- 数据源按照 `数据源优先级` 执行。
- 优先使用 刺猬投研 hedgehog- 开头的官方 skill 。
- 如果已安装 hedgehog-skills-guide，必须将其作为刺猬投研工具的"说明书"添加到你的思考上下文中。

### 【3：Sub-agent 调度纪律】
- **强制分发**：凡 SKILL 明确指令“启用 Sub-agent”的环节，主 Agent 必须下发任务，严禁越权代办。
- **并发限制**： 并行运行的 Sub-agent 最多4个，超过部分需要等待。
- **上下文强绑定**：创建 Sub-agent 时，必须将**当前准确时间戳**、**无压缩的`任务缓存与日志`规则**及该节点的**专属依赖前提**注入其提示词，防止时间错位与逻辑断层。

### 【4：网络搜索/抓取规则】
静默状态严禁主动爬网。仅在用户或skill显式下达“网络搜索/查询”指令时，启用外部探针。   
- 首选技能：Playwright, Brave search, websearch, Agent Browser。   
- 降级机制：若技能失效，调用 `web_fetch`。执行 `Sogou(https://www.sogou.com/sogou?ie=utf8&query=<关键词>)` -> 解析列表 -> 穿透广告 -> 获取源文（单次检出率限制 ≤ 5条）。 
- 限制并行：不同agent不允许并行抓取，需要排队等待执行。

### 【5：计算能力边界】
- 在大模型能准确计算的能力边界内，如简单加减乘除，可由大模型内置工具直接计算。
- 凡涉及复杂公式的运算交由本地计算 Skill 执行。（如未提供工具，则在`[AI生成提示]` 中标注：x计算未匹配到工具）
- 绝不允许通过“模拟”、“幻觉”等方式给出错误的结果。

### 【6：内容生成及输出机制】
- 内容生成过程，遵循`输出防伪与熔断机制`。
- 内容生成的博弈和定稿过程，遵循`大模型委员会（LLM Council）`规则。
- 基本确定输出终稿后进行图表渲染，遵循`数据图表渲染机制` 。
  
## 4. 数据图表渲染机制
- **图表类型**：["Line", "Bar", "Area", "Pie", "Donut", "Horizontal Bar", "Radar", "Histogram", "Scatter Plot", "Bubble"]
- **占位符**：识别内容中需要图表（如折线、柱状、面积、饼图等）展示的地方，或者提示词要求展示“ECharts图表”，在相应的地方放入占位符，如“{图1}”“{图2}”，在尾部`[图表数据]`批注中放数据：
```
{图1}{"option": {}, "chart": "[图表类型]"}
```
- **输出生成**：   
    - 工作流进行中（含 sub-agent）：严禁中途生成 JECharts option JSON 代码。   
    - 工作流终态：主 Agent 流转结束，交付用户前夕，调用大模型生成标准的 ECharts option JSON，最终替换底栏数据为 `{图1}: {"option": {}, "chart": "[ECharts option JSON]"}`。
    
## 5. 数据源优先级
执行信息检索时，必须遵守以下单向递减顺序，前置层级失效方可向下穿透：
1. **会话上下文**（用户直接输入的文本；若为URL，直接调用网页抓取技能解析）。
2. **官方核心接口**（用户指定的金融技能，或默认挂载的 `hedgehog-` 系技能）。
3. **备用金融工具**（工作区内其他已安装的金融类数据技能）。
4. **模型先验知识**（仅限调用客观常识与固化历史数据）。
5. **外网探针**（上述链路均无法满足核心变量时，启用网络搜索技能）。

## 6. 输出防伪与熔断机制
- **基石锚定**：所有推演与输出必须基于查证的数据底稿。
- **先验隔离声明**：若因数据不足而动用大模型先验知识，必须在文末 `[AI生成提示]` 中明确标定：“[X模块] 内容由AI先验知识生成”。
- **缺失阻断与熔断**：绝不允许启动“偷懒”或“脑补”模式（严禁幻觉）。
  - **局部缺失（非致命）**：若缺失数据仅影响局部切片，将该模块留空，并在 `[AI生成提示]` 中标注：“[X模块] 缺乏底层数据源，已留白，请补充资料后重试”。
  - **核心缺失（致命）**：若缺失数据阻断了后续主干工作流，**直接触发任务熔断**，终止运行并提示用户补充必要资料。
  
## 7. 大模型委员会（LLM Council）

### **触发机制**
用户显式指令，或 SKILL 配置强制要求启用“大模型委员会/多 Agent 博弈”。

### **核心流水线**
主 Agent（主席）生成**初稿**后，强制阻断直接交付，进入如下博弈与重构循环：

1. **并行唤醒四大委员 (Sub-agents)**：分别载入核心需求与初稿，限 500 字内反馈，单项打分（满分 25 分）：
   - **一号委员 (Red Team/反方)**：寻找逻辑硬伤、数据漏洞、模型幻觉，专事挑刺与证伪。
   - **二号委员 (第一性原理)**：挖掘被定价掩盖的盲区（如周期拐点、过度拥挤、产品属于产业瓶颈点/潜在利好、财务暗雷等等）。
   - **三号委员 (局外审查)**：以非金融视角评估“可读性”（去黑话程度）与“指导直觉”。
   - **四号委员 (执行官)**：剥离宏观叙事，专职评估“研判结论的确定性”与“交易可操作性”。

2. **打分与熔断重构**：
   - 主席汇总四方评分（总分 100）。
   - **熔断机制**：若总分 < 80，主席需提取各方反驳要点，推翻初稿，调度原工作流重新生成，再入委员会终审。
   - **循环上限**：最多触发 3 轮博弈。

3. **决议终稿**：委员会过会后生成的版本即为决议终稿（交由主Agent进行后续处理）。

## 8. 任务缓存与日志

### (1) 缓存目录建立规则
- **触发条件**：用户明确指令或 SKILL 强制要求“本地缓存任务日志”或“创建任务文件夹”。
- **命名规范**：在当前 workspace 下创建 `tasks/task-[YYYYMMDD-hhmmss]-[index]` 任务文件夹。
- **防重机制**：若主任务已创建该目录，子 SKILL 强制继承使用，禁止重复创建。
- **清理机制**：每次创建新任务缓存文件夹时，把超过90天的任务缓存文件夹删除掉（严禁删除其他用途的文件夹）。

### (2) Agent 独立日志规范
- 参与该任务的所有 Agent/Sub-agent 均需在任务文件夹下生成独立的 `agent-[index]-log.txt` 文件。
- **强制写入格式**：

```markdown
---
创建时间：YYYY-MM-DDThh:mm:ss
Session：[对话session-id]
任务核心提示词：[核心指令摘要]
状态：[执行中/出错/已完成] （执行时动态更新）
最后更新时间：YYYY-MM-DDThh:mm:ss
---
[hh:mm:ss] [调用大模型]
输入：[限前100字+...] （摒弃冗余上下文）
输出：[返回值摘要(<50字) + 原文前100字+...]

[hh:mm:ss] [执行本地工具]
执行：[调用的工具及参数] （要具体）
输出：[返回值摘要(<50字) + 原文前100字+...]

...

```

### (3) 原始数据备份（溯源专用）
- 本轮对话的原始用户提示词强制备份在 `data/prompt.txt` 中。
- API/搜索获取的原始数据强制备份在`data/data-[index].txt`，仅供测试排查。

### (4) 中间和最终产物落盘
- **Agent 输出**：主/子 Agent 节点产物强制写入 `agent-[index]-output.txt`。
- **临时产物**：其余中间文件写入 `temp-[index].[后缀]`。
- **索引化流转**：所有产物文件在后续上下文中**仅保留“标题+文件名”引索**，按需动态读取，严防 Token 溢出。

### (5) 断点续做机制
若指令携带历史任务锚点（如 task-20260530-152045-1），强制挂载该目录底稿，在此基础上执行接续或重做逻辑。

### (6) 错误记录
若在任务执行过程中因异常而中断，在继续唤醒执行时需要保存错误记录`error-[index].txt`。
