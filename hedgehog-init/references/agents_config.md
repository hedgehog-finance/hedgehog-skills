# hedgehog-app增加的设定
version: 1.2.18

## 1. 市场类别（Market_Type）和股票代码规则
**市场矩阵**：
`CN`(A股)：`SH`(上交所)、`SZ`(深交所)、`BJ`(北交所)。 
`CNHK`(港股)：`HK`(港交所，联交所)。 
- **代码规范**：标准格式为 `[代码].[交易所Key]`（如 `600000.SH`）。 
- **硬转换逻辑（禁调外部API）**：遇简码按以下规则强制本地映射：   
    - `SH`：600/601/603(主板)；688(科创板)；110/113/118(可转债)；51/56(ETF)。   
    - `SZ`：000/002(主板)；300(创业板)；123/127/128(可转债)；15(ETF)。   
    - `BJ`：920(北交所)；83/87(可转债)。   
    - `HK`：0(主板)；3/8(创业板)；2(可转债)。 
- **代码查名称**：仅有标准代码时，调系统注册方法查基本信息。 
- **名称查代码**：仅有名称时，强制调 `hedgehog-company-index-data` 的 Tool-1 查验标准代码。 

## 2. 上下文变量与记忆级联 
处理 JSON 请求执行具体动作前，强制按优先级读取并应用以下变量： 
- `cw_system_prompt`：系统级铁律（最高权重）。 
- `cw_market`：市场类别。输入“A股/中国股市/markets.CN”一律折叠为 `CN`（默认），决断底层路由逻辑。 
- `cw_context`：本次会话外部知识（优先采信）。 
- `cw_memory`：本次会话用户记忆。 
- `cw_content`：核心动作指令。 
- `cw_output`：末端输出格式校验锚点。

## 3. 核心调度纪律

### 【1：上下文拦截与格式规范】
遇 JSON 数据请求，按 `上下文变量与记忆级联` 解析业务背景、记忆、指令及 `cw_output` 输出要求（尤指图表混排逻辑）。

### 【2：工作流编排纪律】
- 严格按照最严谨的路径去编排工作流，不允许偷懒，不允许走捷径，不需要你为我省时间和省token。
- 如果你偷懒使用编造的数据，或者使用先验知识替代我的数据，将对我产生灾难性后果。

### 【3：数据源及底层技能路由】
- 严格遵循 `数据源优先级` 执行。
- 优先调用 `hedgehog-` 开头官方 skill。
- 若安装 `hedgehog-skills-guide`，强制将其作为“说明书”载入思考上下文。

### 【4：网络搜索/抓取规则】
静默期严禁主动爬网。仅在用户或skill显式下达“网络搜索/查询”指令时启用探针。   
- 首选技能：Playwright, Brave search, websearch, Agent Browser。   
- 降级机制：技能失效调 `web_fetch`。执行 `Sogou(...)` -> 解析列表 -> 穿透广告 -> 获取源文（单次检出 ≤ 5条）。 

### 【5：计算能力边界】
- 数学表达式计算、复杂公式强制交由本地计算器 Skill 执行（未匹配到工具则在 `[AI生成提示]` 中标注）。
- 严禁通过“模拟”或“幻觉”捏造错误结果。

### 【6：内容生成及输出机制】
- 内容生成必须遵循 `输出防伪与熔断机制`。
- 在用户或skill显式下达指令启用 `大模型委员会（LLM Council）`，则必须遵循。
- 内容在最后交付前渲染图表必须遵循 `数据图表渲染机制`。
  
## 4. 数据图表渲染机制
- **图表类型**：["Line", "Bar", "Area", "Pie", "Donut", "Horizontal Bar", "Radar", "Histogram", "Scatter Plot", "Bubble"]
- **占位符**：识别需图表展示处（或提示词要求 ECharts），插入占位符如 `{图1}`，尾部 `[图表数据]` 放数据列表：
  `{图1}{"option": {}, "chart": "[图表类型]"}`
- **输出生成**：   
    - 工作流中（含 sub-agent输出结果）：严禁中途生成 ECharts option JSON。   
    - 工作流终态：主 Agent 交付前夕调用大模型生成标准 JSON，替换底栏数据为 `{图1}: {"option": {}, "chart": "[ECharts option JSON]"}`。

## 5. 数据源优先级
检索严格遵循单向递减穿透顺序，前置失效方可向下穿透：
1. **会话上下文**（用户直接输入；若为URL直接调网页抓取解析）。
2. **官方核心接口**（指定金融技能或 `hedgehog-` 技能）。
3. **备用金融工具**（工作区内其他已安装技能）。
4. **模型先验知识**（仅限客观常识与固化历史）。
5. **外网探针**（上述皆空时，启用网络搜索）。

## 6. 输出防伪与熔断机制
- **基石锚定**：所有推演与输出强制基于查证底稿。
- **先验隔离声明**：动用先验知识必须在文末 `[AI生成提示]` 标定：“[X模块] 内容由AI先验知识生成”。
- **缺失阻断（严禁幻觉）**：
  - **局部缺失（非致命）**：模块留空，在 `[AI生成提示]` 标注：“[X模块] 缺乏底层数据源，已留白，请补充资料后重试”。
  - **核心缺失（致命）**：阻断主干时**直接触发任务熔断**，终止并提示用户补充资料。

## 7. Sub-agent 调度与验收纪律
通过 `sessions_spawn` 派发子任务时强制遵守：

1. **禁止越权**：主 Agent 严禁代劳，必须下发执行，并记录活跃的`runId`。
2. **上下文强注入**：`task` 描述必须【自包含】。强制打包注入：AGENTS.md 中的“hedgehog-app增加的设定”、核心需求、专属依赖数据及准确时间戳，严防逻辑断层。
3. **交付约束**：Sub-agent 输出必须严守`图表渲染`、`防伪熔断`与`资料引用`纪律，且必须显式返回 `session_id` 、落盘文件名 `agent-[index]-output.txt`及日志文件名 `agent-[index]-log.txt`。
4. **闭环验收**：主 Agent 必须核验 Sub-agent 是否按规生成日志文件，验证项缺失直接打回重做。
5. **滚动执行子任务（双线程滑动窗口）**：处理子任务列表时，须严格按以下状态机循环执行直至清空：
    - 步骤1. **核验水位**：检查当前活跃的 Sub-agent 数量（活动队列 `runId` 计数）。
    - 步骤2. **派发补齐 (Spawn)**：若活跃数 < 2 且任务列表有余量，立即取出新任务调用 `sessions_spawn`，并将返回的 `runId` 压入活动队列，补满 2 个并发。
    - 步骤3. **挂起让权 (Yield)**：若队列已达 2 个，或任务列表已空但队列仍有残留，**强制调用 `sessions_yield()` 挂起主会话**。
    - 步骤4. **回收与滚动 (Loop)**：被 `[Subagent Outcome]` 推送唤醒后，读取汇总数据并从队列剔除该 `runId`。**严禁此时执行后续主流程**，必须立即重返【步骤1】补齐并发空位后再次挂起。
    - 步骤5. **终局与熔断**：仅当【任务列表为空】且【活动队列为 0】时跳出循环，整合数据执行后续主流程。若循环超 50 次，触发死循环熔断，强制清空队列并终止滚动。

## 8. 大模型委员会（LLM Council）

### **触发机制**
用户显式指令或 SKILL 强制要求，在数据图表渲染前触发。

### **核心流水线**
主 Agent 生成**初稿**后阻断交付，进入博弈重构循环：
1. **使用 sessions_spawn 并行唤醒四大委员 (Sub-agents)**：载入需求与初稿，返回结果（500 字内反馈；单项评分[满分 25 分])：
   - **一号委员 (Red Team/反方)**：找逻辑硬伤、数据漏洞、模型幻觉，专事挑刺证伪。
   - **二号委员 (第一性原理)**：挖定价盲区（周期拐点、过度拥挤、产业瓶颈/潜在利好、财务暗雷）。
   - **三号委员 (局外审查)**：非金融视角评估“可读性”（去黑话）与“指导直觉”。
   - **四号委员 (执行官)**：剥离宏观叙事，评估“研判确定性”与“交易可操作性”。
2. **打分与熔断重构**：
   - 使用 sessions_yield 挂起，等待所有委员返回结果。
   - 主席汇总总分（满分 100）。
   - **熔断机制**：总分 < 75，提取反驳要点，推翻初稿重新生成，再入终审。
   - **循环上限**：最多 3 轮博弈。
3. **决议稿**：过会版本即为决议稿，交主 Agent 继续处理。

## 9. 任务缓存与日志

### (1) 缓存目录建立规则
- **触发条件**：明确指令或 SKILL 强制要求“本地缓存/创建任务文件夹”。
- **命名规范**：当前 workspace 下建 `tasks/task-[YYYYMMDD-hhmmss]-[index]`。
- **防重机制**：主任务已建则子 SKILL 强制继承，禁重复创建。
- **清理机制**：新建时静默删除超 90 天的任务缓存（严禁误删其他文件夹）。

### (2) Agent 独立日志规范
- 参与任务的所有 Agent/Sub-agent 在目录下生成 `agent-[index]-log.txt`。
- **强制写入格式**：

```markdown
---
创建时间：YYYY-MM-DDThh:mm:ss
Session：[对话session-id]
任务核心提示词：[核心指令摘要]
状态：[执行中/出错/已完成] （动态更新）
最后更新时间：YYYY-MM-DDThh:mm:ss
---

[hh:mm:ss] [调用大模型]
输入：[限前100字+...] （摒弃冗余上下文）
输出：[返回值摘要(<50字) + ...]

[hh:mm:ss] [执行本地工具]
执行：[调用的工具及具体参数] 
输出：[返回值摘要(<50字) + ...]
```

### (3) 原始数据备份（溯源专用）
- 本轮对话原始提示词强制备份至 `data/prompt.txt`。
- API/搜索原始数据强制备份至 `data/data-[index].txt`，仅供排查。

### (4) 中间和最终产物落盘
- **Agent 输出**：主/子 Agent 节点产物强制写入 `agent-[index]-output.txt`。
- **临时产物**：其余写入 `temp-[index].[后缀]`。

### (5) 错误记录
- 异常中断时唤醒执行需保存 `error-[index].txt`。

### (6) 消耗token统计
- 主Agent在所有Sub-agent完成后，用session_status(sessionKey=<sub-agent-sessionKey>)逐个回查token用量，并追加记录到`error-[index].txt`文件中。
- 主Agent在终稿交付前，用session_status(sessionKey="current")读取自己的用量，追加记录到`error-[index].txt`文件中。

## 10. 资料引用格式
资讯分类：`{'News':'新闻', 'Research':'研报', 'Announcements':'公告', 'Minutes':'纪要', 'Views':'观点'}`。
- 内部资讯：`{[资讯分类]:[id]} [标题]` (如: `{新闻:12345} 统计局...`)。
- 本地文件：`{file:[workspace路径及文件名]} [标题(可选)]` (如: `{file:test.txt} 6月5日快讯`)。
- 外部资料：`{url:[url]} [标题(可选)]` (如: `{url:https://ciweiai.com} 刺猬投研AI`)。
